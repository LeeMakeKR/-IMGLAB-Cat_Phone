## 2026-04-10 — 코드 점검 (v4)

### 🔴 치명적 문제

#### 1. 버튼 핀 10 — SPI SS 핀 충돌
- **위치**: `Cat_Phone.ino` L46 (`#define BUTTON_PIN 10`), L149
- **원인**: Arduino UNO의 핀 10은 SPI 하드웨어 SS(Slave Select) 핀. `sd.begin()` / `MP3player.begin()` 내부에서 `SPI.begin()`이 호출될 때 핀 10이 강제로 OUTPUT으로 변환됨.
- **증상**: `digitalRead(BUTTON_PIN)`이 정상 작동하지 않아 전체 상태 머신이 작동 불능 상태가 됨.
- **해결**: 버튼 핀을 SPI와 무관한 핀(예: 2, 6, 7, 8, A1)으로 변경.
- **상태**: ⬜ 미해결

### 🟡 중간 수준 문제

#### 2. VS1053 초기화 실패 시 계속 실행
- **위치**: `Cat_Phone.ino` L166–L170
- **원인**: `MP3player.begin()`이 0이 아닌 값을 반환해도 에러 출력 후 그대로 진행.
- **증상**: 이후 `playTrack()`, `isPlaying()` 등 호출 시 예측 불가한 동작 또는 hang 발생 가능.
- **해결**: 초기화 실패 시 halt 처리 또는 재시도 로직 추가.
- **상태**: ⬜ 미해결

#### 3. SD 카드 초기화 실패 시 복구 불가 halt
- **위치**: `Cat_Phone.ino` L161–L163
- **원인**: `sd.initErrorHalt()`는 프로그램을 영구히 멈춤.
- **증상**: 전시/설치물 환경에서 SD 오류 발생 시 수동 리셋 없이는 복구 불가.
- **해결**: 소프트 재시도 루프 또는 watchdog 타이머 활용.
- **상태**: ⬜ 미해결

#### 4. STATE_PLAYING 상태에서 재생 완료 미감지
- **위치**: `Cat_Phone.ino` L242–L250
- **원인**: MP3 재생이 끝나도 상태가 `STATE_PLAYING`에 유지됨.
- **증상**: 수화기를 계속 들고 있으면 재생 종료 후 시스템이 아무 동작도 하지 않는 상태에 영구히 머묾.
- **해결**: `!MP3player.isPlaying()` 감지 시 `STATE_IDLE` 또는 적절한 상태로 전환하는 로직 추가.
- **상태**: ⬜ 미해결

### 🟢 경미한 문제

#### 5. 설정값에 `const` 누락
- **위치**: `Cat_Phone.ino` L31–L39
- **원인**: `WAIT_TIME_MS` 등 상수 목적의 변수가 `const` 없이 선언됨.
- **증상**: AVR에서 Flash 대신 SRAM에 저장되어 RAM 낭비 (Arduino UNO는 2KB RAM).
- **해결**: `const unsigned long`, `const uint8_t` 등으로 선언.
- **상태**: ⬜ 미해결

#### 6. SD 카드 내 macOS 메타데이터 파일
- **위치**: `mp3/` 폴더
- **내용**: `._track001.mp3` ~ `._track004.mp3` (macOS 리소스 포크 파일)
- **증상**: `playTrack()` 번호 기반 재생에는 직접 영향 없으나, 일부 라이브러리 버전에서 파일 열거 시 오작동 가능.
- **해결**: SD 카드에서 `._*` 파일 삭제.
- **상태**: ⬜ 미해결

#### 7. 시리얼 로그 트랙 번호 포맷 오류 가능성
- **위치**: `Cat_Phone.ino` L111–L112
- **원인**: `"[MP3] track00"` + track 번호를 문자열 연결로 출력. `TRACK_COUNT`가 10 이상이면 `track0010` 같은 잘못된 포맷 출력.
- **증상**: 현재 4개이므로 문제 없으나, 트랙 수 확장 시 로그 오독 가능.
- **해결**: 트랙 번호 자릿수에 맞는 포맷 함수 사용.
- **상태**: ⬜ 미해결

## SD 카드 포맷 규격

| 용량 | 권장 파일시스템 |
|------|---------------|
| 2GB 이하 | FAT16 |
| 4GB ~ 32GB | **FAT32** |
| 64GB 이상 | FAT32 강제 포맷 필요 (기본 exFAT 불가) |

- Windows 포맷 설정: 파일 시스템 **FAT32**, 할당 단위 크기 **32KB**
- `SdFat` 라이브러리는 FAT16/FAT32만 지원
